## ADDED Requirements

### Requirement: 专辑详情页支持头部与 tab 内容区的连续滚动接力
系统 SHALL 让专辑详情页的外层头部与当前 tab 的内容列表形成连续滚动链路，而不是让外层列表和内层列表彼此抢占手势或通过硬开关切换滚动能力。

#### Scenario: 上滑时优先折叠头部再滚动当前 tab 内容
- **WHEN** 用户在专辑详情页执行纵向上滑手势
- **THEN** 系统先让外层头部继续折叠
- **AND** 仅在头部已无法继续上滑消费后，才把剩余位移交给当前 tab 的内层列表

#### Scenario: 下滑时先让当前 tab 回顶再展开头部
- **WHEN** 用户在专辑详情页执行纵向下滑手势
- **THEN** 系统先让当前 tab 的内层列表回到顶部
- **AND** 只有在当前 tab 已经无法继续向下回滚时，才用剩余位移展开外层头部

#### Scenario: 横向手势优先交给 pager
- **WHEN** 用户在专辑详情页的 tab 内容区执行以横向分量为主的手势
- **THEN** 系统保持该手势用于 `HorizontalPager` 切页
- **AND** 不因为轻微的纵向偏移而过早抢占为纵向滚动

#### Scenario: 切换 tab 后保留各自独立的滚动位置
- **WHEN** 用户在专辑详情页于“歌曲”和“简介”两个 tab 之间切换
- **THEN** 系统保持外层头部当前折叠状态不被重置
- **AND** 为每个 tab 分别保留其内层列表滚动位置

## MODIFIED Requirements

### Requirement: 系统提供专辑详情页承载专辑内容、简要百科信息、歌曲列表与动态信息
系统 SHALL 提供可通过专辑实体标识直接进入的专辑详情页，并使用“获取专辑内容”接口加载专辑基础信息、简要百科信息与歌曲列表，同时使用“专辑动态信息”接口加载评论数、分享数、收藏数等动态元信息。专辑详情页 MUST 以头部 hero、sticky tab 与 tab pager 的形式组织这些内容，而不是继续把简介和歌曲列表平铺在同一条外层列表中。

#### Scenario: 从搜索结果进入专辑详情页
- **WHEN** 用户点击任一专辑搜索结果项
- **THEN** 系统打开对应专辑详情页
- **AND** 详情页使用该结果项携带的专辑实体标识请求“获取专辑内容”接口
- **AND** 详情页使用同一专辑实体标识请求“专辑动态信息”接口

#### Scenario: 专辑内容接口成功返回后展示头部、歌曲 tab 与简介 tab
- **WHEN** 专辑详情页成功获得可解析的专辑内容数据
- **THEN** 系统展示专辑封面、专辑标题、歌手、公司、发行信息或可用的辅助说明信息
- **AND** 系统在头部下方展示至少两个 tab：`歌曲` 与 `简介`
- **AND** 系统默认进入 `歌曲` tab 承载歌曲列表和相关操作
- **AND** 系统在 `简介` tab 中承载完整专辑简介正文，而不是在外层列表额外追加独立简介卡片

#### Scenario: 专辑动态信息接口成功返回后展示在 tab 上方的头部区域
- **WHEN** 专辑详情页成功获得专辑动态信息数据
- **THEN** 系统在头部区域底部、sticky tab 上方展示评论数、分享数、收藏数或可解析的动态元信息
- **AND** 该动态元信息与头部一起滚动，而不是留在任一 tab 的内容列表里
- **AND** 头部动作区可以展示评论数摘要胶囊，但当前版本不要求该胶囊具备评论页跳转能力

### Requirement: 专辑详情页视觉层次偏向云音乐风格
系统 SHALL 让专辑详情页采用更偏云音乐的音乐内容详情视觉层次，包括带氛围感的头部背景、圆角专辑封面、紧凑的信息排版、sticky tab、可区分的动态元信息区块与更接近音乐列表的歌曲条目密度，而不是通用卡片流样式。

#### Scenario: 专辑详情页展示云音乐风格头部与 tab 内容区
- **WHEN** 用户查看专辑详情页
- **THEN** 系统在页面顶部展示具有氛围感的专辑详情头部
- **AND** 返回入口固定在详情页外层且不随正文滚动
- **AND** 专辑主视觉以圆角方形封面样式展示
- **AND** 头部氛围色优先依据专辑封面主色生成渐变效果
- **AND** 当封面缺失或取色失败时退回稳定的默认渐变
- **AND** 头部底部使用从黑色半透明到透明的渐变遮罩，确保白色标题和摘要在复杂背景上仍可读
- **AND** 头部不重复展示完整或摘要版简介正文，避免与 `简介` tab 内容重复
- **AND** 头部不再提供单独的“查看简介”动作按钮
- **AND** 专辑动态信息卡展示在头部底部、sticky tab 上方，并与头部共同滚动
- **AND** sticky tab 与 pager 内容区在视觉上保持连续承接
- **AND** 歌曲列表继续以更接近音乐 App 的紧凑行样式承载，并且不重复展示专辑封面缩略图
