## MODIFIED Requirements

### Requirement: 在线播放支持当前项优先的有限预热
系统 SHALL 支持对可缓存在线播放资源执行有限预热，以提前缓存当前播放点之后的一段数据或下一首可播放首段；预热 SHALL 有明确预算，不得默认整首缓存。系统 MUST 优先保障当前播放项缓存，不得在当前播放项仍需要网络缓存推进时为下一首抢占预热带宽。系统 MUST 不因当前网络为移动网络或非 Wi-Fi 而跳过可缓存目标的预热。

#### Scenario: 下一首在线歌曲被有限预热
- **WHEN** 当前播放队列存在下一首在线歌曲
- **AND** 预热功能已开启
- **AND** 当前播放项已经完整缓存或被判定不需要继续写入缓存
- **THEN** 系统为下一首歌曲解析可缓存资源并预热首段数据
- **AND** 预热在达到配置的时长或字节预算后停止
- **AND** 系统不得为了预热直接缓存整首歌曲

#### Scenario: 当前播放项未缓存完成时不预热下一首
- **WHEN** 当前在线播放歌曲仍在推进自身缓存
- **AND** 当前播放项尚未完整缓存
- **THEN** 系统不启动下一首歌曲预热
- **AND** 当前网络读取优先用于当前播放项的播放和缓存

#### Scenario: 当前播放项维持 ahead window
- **WHEN** 当前在线播放歌曲正在播放
- **AND** 预热功能已开启
- **THEN** 系统在当前播放位置之后维持有限预读窗口
- **AND** 该窗口写入缓存后可被当前项缓存进度投影复用

#### Scenario: 移动网络不跳过当前项预热窗口
- **WHEN** 当前在线播放歌曲正在播放
- **AND** 预热功能已开启
- **AND** 当前网络为移动网络
- **THEN** 系统仍可为当前播放位置之后的 ahead window 执行有限预热
- **AND** 预热进度通过独立预热快照发布
