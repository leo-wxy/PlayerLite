## MODIFIED Requirements

### Requirement: 设置页展示并管理当前播放偏好
系统 SHALL 在设置页集中展示并管理当前播放偏好，包括歌曲缓存上限、默认音质、当前可用音源列表，以及在线播放预热偏好，而不是只把这些信息保留在隐式本地存储中。系统 MUST 不再展示或管理“仅 Wi-Fi 自动缓存”和“移动网络只播放不缓存”这类网络类型缓存限制偏好。

#### Scenario: 打开设置页时看到预热偏好
- **WHEN** 用户进入设置页
- **THEN** 系统在播放与缓存分组展示在线播放预热开关
- **AND** 系统展示当前预热预算摘要
- **AND** 文案明确预热是有限首段缓存，不等同于整首下载

#### Scenario: 打开设置页时不展示网络类型缓存限制
- **WHEN** 用户进入设置页
- **THEN** 系统不得展示仅 Wi-Fi 自动缓存开关
- **AND** 系统不得展示移动网络只播放不缓存开关
- **AND** 移动网络下的在线播放缓存能力不需要用户额外开启

#### Scenario: 用户修改预热开关
- **WHEN** 用户在设置页切换在线播放预热开关
- **THEN** 系统持久化新的开关值
- **AND** 设置页立即展示新的开关状态
- **AND** 可立即作用于播放进程的预热策略被下发到播放进程

#### Scenario: 用户修改预热预算
- **WHEN** 用户在设置页选择新的预热预算
- **THEN** 系统持久化新的预热预算
- **AND** 后续预热任务使用新的时长或字节上限
- **AND** 已经基于旧预算运行的预热任务会被取消或按新预算重新调度

#### Scenario: 设置页重建后恢复预热偏好
- **WHEN** 设置页因进程或 Activity 重建重新加载
- **THEN** 系统从播放偏好存储恢复预热开关和预算
- **AND** 不会回退到与用户上次保存值不一致的默认状态
