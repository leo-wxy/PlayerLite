## ADDED Requirements

### Requirement: 用户中心已登录内容区通过 Tab 切换展示个人收藏内容
系统 SHALL 在用户中心的已登录内容区提供与头部资料区分层的 Tab 内容面板，并通过 Tab 切换展示收藏的歌手、收藏的专栏和用户歌单；该内容区 MUST 延续账户域偏云音乐的个人页视觉风格，而不是继续以单段静态占位说明代替真实内容。

#### Scenario: 已登录用户进入用户中心时看到 Tab 内容区
- **WHEN** 用户已登录并进入用户中心页面
- **THEN** 系统展示头部资料区下方的 Tab 切换栏与内容面板
- **AND** Tab 切换栏按“用户歌单、收藏歌手、收藏专栏”的顺序提供三个入口
- **AND** 系统默认选中“用户歌单”并展示对应内容，而不是将三个内容区同时堆叠展示
- **AND** Tab 栏与当前内容列表在视觉上保持连续的同一内容区，而不是分裂成上下两段间距过大的独立面板
- **AND** 头部资料卡在已登录态保持独立卡片表现和完整圆角，中间不再插入额外的独立说明卡
- **AND** 资料卡与下方内容面板之间保持小幅分组间距，而不是完全连成一块或被过大间距拉开
- **AND** 在移动端页面宽度下，资料卡与内容面板仅保留小幅横向留白，而不是继续使用过大的默认边距或完全贴边

#### Scenario: 用户切换 Tab 时仅更新当前内容面板
- **WHEN** 用户在用户中心点击另一个收藏内容 Tab
- **THEN** 系统切换到对应的内容面板
- **AND** 用户中心头部资料区与页面主结构保持稳定，不因 Tab 切换而重建为另一套页面

#### Scenario: 当前内容列表使用紧凑列表节奏展示
- **WHEN** 用户浏览任意一个已加载的收藏内容 Tab
- **THEN** 当前内容列表中的 item 采用紧凑的垂直间距
- **AND** Tab 区与首个内容 item 之间不应保留明显割裂的大间距

#### Scenario: 内容区上滑时筛选 Tab 保持吸顶
- **WHEN** 用户在已登录的个人中心持续向上滑动内容区
- **THEN** 顶部筛选 Tab 在资料头部和说明文案滚出后保持吸顶可见
- **AND** 吸顶区域继续与当前内容面板保持连续，而不是把说明文案整块固定在顶部

#### Scenario: 当前 Tab 没有内容时展示空状态
- **WHEN** 当前选中的收藏内容 Tab 请求成功但返回空内容
- **THEN** 系统在当前内容面板展示对应空状态说明
- **AND** Tab 切换栏与其他已加载内容保持可用

#### Scenario: 当前 Tab 请求失败时展示局部重试
- **WHEN** 当前选中的收藏内容 Tab 请求失败且不属于会话失效
- **THEN** 系统仅在当前内容面板展示错误提示与重试入口
- **AND** 用户中心头部资料区与 Tab 切换栏继续保持可用
