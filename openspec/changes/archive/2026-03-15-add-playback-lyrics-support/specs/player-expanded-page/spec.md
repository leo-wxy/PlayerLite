## MODIFIED Requirements

### Requirement: 顶栏歌曲与歌词切换保持稳定且小屏可用
系统 SHALL 在播放展开页顶栏提供“歌曲”和“歌词”两个并列 tab，用于在歌曲主界面与歌词主界面之间切换；歌曲页保留歌名、歌手与当前行歌词摘要，歌词页则作为顶栏下方的完整沉浸页面承载歌词滚动视图，并与歌曲页共享稳定的小屏布局节奏。

#### Scenario: 歌曲页信息固定在底部信息区
- **WHEN** 用户查看当前播放页
- **THEN** 歌名与歌手固定展示在底部信息区
- **AND** 歌手下方展示当前时间对应的一行歌词摘要，或在无可用歌词时展示稳定空态说明
- **AND** 歌曲信息和歌词状态切换不会把底部整体布局来回抖动

#### Scenario: 顶栏歌词 tab 切换到完整沉浸歌词页
- **WHEN** 用户查看当前播放页
- **THEN** 系统在顶栏“歌曲”旁边提供“歌词”tab
- **AND** 顶栏 tab 以轻量文本导航和细指示线表达选中状态，而不是使用厚重胶囊式按钮
- **AND** 用户切换到“歌词”tab 后进入与歌曲页同级的完整歌词页面
- **AND** 歌词页不再显示歌名、歌手头部信息
- **AND** 歌词页不再显示进度区或播控区
- **AND** 歌词页不使用卡片、边框或方框式容器包裹歌词内容
- **AND** 当前歌词摘要与完整歌词视图保持同一首歌曲上下文

#### Scenario: 用户可通过上半区横滑在歌曲页与歌词页之间切换
- **WHEN** 用户在播放页上半区主内容区域执行水平滑动
- **AND** 当前位于歌曲页或歌词页
- **THEN** 系统允许用户通过左滑/右滑在两个同级页面之间切换
- **AND** 顶栏 tab 选中状态与当前页面保持同步
- **AND** 横滑手势不占用歌曲页底部进度区的 seek 拖动手势
- **AND** 顶栏不承担横滑切页手势

#### Scenario: 歌词页沿用歌曲页的上下留白节奏
- **WHEN** 用户切换到歌词页
- **THEN** 歌词主内容顶部起始位置与歌曲页封面顶部对齐
- **AND** 歌词页左右内边距与歌曲页主内容保持一致
- **AND** 歌词页底部留白与歌曲页播控距底部的视觉基线保持一致
- **AND** 最后几行歌词不会贴底或悬在中部

#### Scenario: 歌词主界面自动滚动到当前行
- **WHEN** 用户位于歌词主界面
- **AND** 当前歌曲存在已解析歌词与当前命中行
- **THEN** 系统自动滚动歌词列表，让当前命中行进入可见区域
- **AND** 当前命中行保持高亮，随播放进度继续推进
- **AND** 当前命中行的字号与视觉强调保持克制，不会因过度放大而打断整页阅读节奏
- **AND** 当前命中行可使用轻微辉光或等价高亮来表达聚焦，而周围歌词按与当前行的距离逐级弱化

#### Scenario: 窄屏设备上的播控区整体更靠下且按钮不重叠
- **WHEN** 播放页运行在较窄宽度或较小高度的设备上
- **THEN** 播控区整体更靠下，和进度条左右边距保持一致
- **AND** 播放模式、上一首、下一首和播放列表按钮仍互不重叠

#### Scenario: 进度条采用更接近 progressbar 的细轨道样式
- **WHEN** 用户查看播放页进度区
- **THEN** 系统使用细轨道和小圆点 thumb 的播放器进度条样式
- **AND** 不继续暴露默认表单式 Slider 的粗轨道观感

#### Scenario: 四个侧边播控按钮保持舒适点击尺寸并带稳定背景
- **WHEN** 用户查看播放页播控区
- **THEN** 除中间主播放按钮外的四个侧边播控按钮在不同屏幕上都保持舒适且清晰的点击区域
- **AND** 每个按钮都有稳定背景，便于识别点击范围
- **AND** 此轮以布局节奏调整为主，不额外改写既有按钮皮肤风格

#### Scenario: 中间主播放按钮大于四个侧边按钮但不过分压迫布局
- **WHEN** 用户查看播放页播控区
- **THEN** 中间播放 / 暂停按钮在视觉尺寸上大于四个侧边按钮
- **AND** 主按钮与侧边按钮保持克制且协调的比例，不会显得失衡或压过整排播控
- **AND** 用户可以一眼识别它是主操作入口
