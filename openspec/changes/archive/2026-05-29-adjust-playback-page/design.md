## Context

本次变更聚焦播放展开页整体 UI 效果。当前规格已经约束了播放页的核心能力：正方形封面、混合封面背景、歌曲信息、歌词摘要、细进度条、音质音效联合状态区、三按钮核心播控、播放模式、播放列表、更多操作、歌曲详情 / 分享 / 收藏等入口。用户确认后的方案不再显示“歌曲 / 歌词”文字 tab，原 tab 区域改为居中歌曲名和歌手名。

用户提供的 `/Users/wxy/Projects/player-lite/aa.html` 是视觉参考稿，偏向沉浸式深色音乐播放器：顶部信息简洁、封面居中突出、底部进度与播控集中、按钮透明或玻璃质感。已整理后的新版 `aa.html` 将这个视觉方向和当前播放页已有能力合并，作为后续 Compose UI 实现的对齐样稿。

## Goals / Non-Goals

**Goals:**

- 让播放页整体观感靠近新版 `aa.html`：深色沉浸背景、同源封面模糊层、玻璃感顶栏按钮、居中封面、底部信息与播控集中。
- 保留现有播放页行为入口，不因视觉调整删除歌词页能力、歌词摘要、音质音效状态、播放模式、列表、更多操作、百科、分享、收藏等能力；但歌曲 / 歌词文字 tab 不再作为顶部可见控件。
- 让小屏布局有明确压缩策略：优先缩小封面和信息区间距，保持播控按钮可点且不重叠。
- 将静态 HTML 参考稿转化为可测试的 OpenSpec 要求，为后续实现提供验收依据。

**Non-Goals:**

- 不调整播放服务、队列、缓存、音频解码、MediaSession 或网络接口。
- 不新增外部 UI 依赖，不把 Tailwind / Font Awesome 引入 Android 实现。
- 不重新设计歌词完整页、更多操作浮层、播放列表 sheet 的业务流程。
- 不改变歌曲详情、收藏、分享、音质、音效等入口的既有行为语义。

## Decisions

### Decision 1: 以新版 `aa.html` 作为视觉样稿，而不是直接照搬原始 HTML

原始 `aa.html` 有沉浸式音乐播放器的视觉方向，但缺少当前产品已有的歌词摘要、音质音效联合状态、百科 / 收藏 / 分享等入口。新版预览稿保留参考稿的整体氛围，同时按用户确认移除顶部“歌曲 / 歌词”可见 tab，并把该区域替换为歌曲名 / 歌手信息，因此后续实现应以确认后的预览稿为准。

备选方案是完全照搬原始 HTML。该方案实现目标更单一，但会让现有播放页规格中的多个入口无处承载，后续再补会造成二次返工。

### Decision 2: 视觉调整只落在播放页 Compose UI 层

本次变更只需要改变布局、颜色、层级、间距、按钮皮肤和状态展示位置，不需要调整 `PlayerRuntime` 的播放状态来源，也不需要改变 playback-service 的跨进程契约。实现时应优先在播放页 UI 组件和局部样式常量中完成。

备选方案是引入新的主题 token 或全局设计系统改造。当前改动目标只针对播放展开页，扩大到全局主题会增加回归范围，并影响首页、详情页、mini player 等无关区域。

### Decision 3: 保留当前交互入口，用视觉层级重新组织

新版播放页应维持当前交互能力，但减少“工具面板感”。顶部栏只保留返回和更多，顶部居中信息区承载歌曲、歌手；主视觉区域承担封面和 buffering 状态；封面下方承载当前歌词摘要；进度区承载时间与音质音效状态；核心播控区只承载上一首、播放 / 暂停、下一首三个按钮；次级动作区承载播放模式、歌曲详情、音效、播放列表、更多等入口。

备选方案是把次级入口都塞进右上角更多操作。该方案页面更干净，但会降低已存在高频入口的可发现性，且与当前规格中的直达入口要求不完全一致。

### Decision 4: 小屏优先保证播控可用，再压缩视觉留白

小屏适配时，优先保证五个核心播控按钮的点击尺寸和间距，其次保证进度条与时间可读，再压缩封面尺寸、封面上方留白和歌词摘要间距。不能通过隐藏核心播控来换取视觉空间。

备选方案是固定视觉比例。该方案在标准屏幕上稳定，但在较矮屏幕上容易让底部按钮贴边、重叠或不可见。

## Risks / Trade-offs

- [Risk] 静态 HTML 和 Compose 渲染能力不同，玻璃模糊、封面背景和阴影效果可能无法逐像素一致。→ Mitigation: 以视觉层级和布局节奏为验收标准，不要求 CSS 效果完全等价。
- [Risk] 播放页已有入口较多，全部保留可能让页面再次变复杂。→ Mitigation: 通过主 / 次级动作分区和透明按钮层级控制注意力，而不是删除功能。
- [Risk] 小屏设备上封面、歌词摘要和播控同时出现时空间紧张。→ Mitigation: 为小屏定义压缩顺序，先缩封面与间距，保持播控和进度区稳定。
- [Risk] 背景取色与封面模糊层如果处理不当，会影响文字对比度。→ Mitigation: 继续使用深色遮罩，并要求前景文字、图标和进度条保持可读。

## Migration Plan

1. 保留新版 `aa.html` 作为视觉参考稿。
2. 更新播放页相关 delta specs，明确需要调整的视觉和布局行为。
3. 后续实现阶段只修改播放页 UI 相关文件。
4. 如实现效果不符合预期，可回退播放页 UI 改动；本变更不涉及数据迁移、接口迁移或持久化变更。
