## Why

当前播放展开页已经具备封面背景、歌词切换、播放控制、音质音效状态和更多操作等能力，但整体视觉参考仍不够集中，页面层级、按钮节奏和沉浸式背景需要进一步统一。

本次调整以 `aa.html` 的沉浸式音乐播放器视觉为参考，并结合当前播放页已有交互，先形成一版可对齐的静态 HTML 效果稿，再据此推进 Compose 播放页 UI 调整。

## What Changes

- 调整播放展开页整体 UI 效果，采用更完整的沉浸式封面背景、深色遮罩、玻璃感顶栏按钮和底部播控层级。
- 顶部不再显示“歌曲 / 歌词”文字 tab，原 tab 位置由当前歌曲名和歌手名承接；保留歌词页作为完整页面能力，不改变其状态与手势切换基础。
- 保留当前播放页已有的封面主视觉、当前歌词摘要、进度条、音质音效联合状态、播放模式、上一首、播放 / 暂停、下一首、播放列表、更多操作等行为入口。
- 将参考 HTML 的标题、封面、进度、主播控和次级操作节奏与现有播放页能力融合，避免只照搬纯静态页面而丢失当前产品功能。
- 优化小屏下封面、歌曲信息、进度区和播控区的垂直节奏，确保按钮不重叠、信息不遮挡、底部操作仍可达。
- 不改变播放服务、队列、音频解码、MediaSession 或网络接口契约。

## Capabilities

### New Capabilities

无。

### Modified Capabilities

- `player-expanded-page`: 调整播放展开页视觉层级、封面背景、信息区、进度区和播控区的展示要求。
- `playback-page-shell`: 调整简洁播放壳层的视觉参考和核心信息布局要求。

## Impact

- 参考稿：`aa.html`
- 主要实现范围预计在 `app/src/main/java/com/wxy/playerlite/feature/player/` 及播放页 Compose UI 相关文件。
- 可能影响播放页视觉布局、触控区域、横竖屏 / 小屏适配和歌词页切换体验。
- 不涉及播放进程、缓存、网络、持久化 schema、Manifest、权限或构建配置。
