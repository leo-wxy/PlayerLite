## Context

当前仓库已经完成了本地/在线统一最小可播放契约，以及“外部页面可替换当前播放列表并按 index 激活曲目”的核心语义。但本地播放入口仍主要依赖系统文件选择器，这意味着：
- 用户只能一次选一首或少量文件，缺少一个长期可见的本地歌曲列表页面。
- 无法验证“再次进入时先展示缓存结果”的体验。
- 无法自然承载“播放全部”“点击某首歌按当前列表 index 播放”和手动重新扫描。

因此，本地歌曲应该从当前在线详情播放 change 中拆出，作为一个单独 change 交付，聚焦个人中心入口、本地歌曲页和扫描缓存链路。

## Goals / Non-Goals

**Goals**
- 在个人中心页提供稳定可见的“本地歌曲”入口，不依赖登录态。
- 新增独立本地歌曲页，承载本地扫描结果、缓存快显、空态/失败态和手动重新扫描。
- 让本地歌曲页的“播放全部”和点击歌曲统一走当前列表替换播放语义。
- 让本地歌曲页在播放请求成功后明确回到主壳层并展开播放器，而不是停留在列表页里没有完成态反馈。
- 复用已有最小可播放契约和外部队列播放入口，不再为本地歌曲页复制播放器同步逻辑。

**Non-Goals**
- 本次不做复杂的本地歌曲分组、搜索、排序筛选或编辑能力。
- 本次不把本地歌曲扫描结果升级为数据库级媒体库；首版只需要稳定的轻量缓存。
- 本次不改变现有系统文件选择器入口，新的本地歌曲页与它并存。

## Decisions

1. 个人中心页新增稳定入口，本地歌曲页使用独立 Activity 承载。
   - 方案：在个人中心资料头部下方、内容面板上方增加“本地歌曲”入口，登录态和游客态都可见。入口打开独立的本地歌曲 Activity，而不是把本地歌曲塞进当前个人中心 tab 的长列表里。
   - 原因：独立页面更适合承载权限态、缓存列表、扫描刷新和播放动作，也能避免把用户中心页继续做重。

2. 本地歌曲数据源采用“MediaStore 扫描 + 本地快照缓存”的双层结构。
   - 方案：Repository 首次扫描 `MediaStore` 生成本地歌曲条目列表，并把结果写入本地快照缓存。再次进入页面时优先展示缓存，再由用户决定是否手动重新扫描。
   - 原因：这样可以满足“再次打开优先展示缓存”的体验，同时避免每次进入都阻塞在一次新的全量扫描上。

3. 右上角扫描入口是显式刷新动作，而不是进入页面自动强刷。
   - 方案：页面首次打开时若有缓存则先展示缓存；右上角提供“扫描”入口，用于用户主动刷新本地歌曲结果。
   - 原因：自动重扫会破坏缓存快显体验，也会让用户误以为页面总是很慢。

4. 本地歌曲页播放能力复用现有外部队列播放入口。
   - 方案：本地歌曲页把扫描结果映射成当前 app 已经使用的 `PlaylistItem` / `LocalMusicInfo` 投影，再通过现有外部队列播放 seam 发起“播放全部 / 点击歌曲播放”。
   - 原因：这样可以直接继承已经验证过的“替换当前列表 + index 激活 + 同列表只切换激活项”语义，而不是为本地页复制一套播放同步逻辑。

5. 本地歌曲页播放成功后通过结果回传让主壳层直接展开播放器。
   - 方案：本地歌曲页 ViewModel 在播放请求成功时发出一次性事件；`LocalSongsActivity` 把该事件转换为 Activity result，`MainActivity` 收到结果后切回 Home 域并展开播放器。若播放请求失败，则保留在当前页并给出短提示。
   - 原因：用户反馈“点击播放事件还没有完成”，问题不在列表点击本身，而在播放成功后缺少明确收口动作。通过结果回传可以让“我的 -> 本地歌曲 -> 点击播放”形成完整闭环，同时保持 `MainActivity` 只负责壳层导航切换。

## Risks / Trade-offs

- [本地歌曲页若每次进入都重新扫描，会破坏缓存快显体验] -> 先展示缓存，重扫仅由显式动作触发。
- [扫描结果缓存过旧时，用户可能误以为列表不完整] -> 通过右上角“扫描”入口和清晰状态文案提示用户手动刷新。
- [无权限时直接把页面当空列表展示，会让用户误以为没有本地歌曲] -> 页面需要单独呈现权限态或可解释失败态。
- [播放请求成功但页面仍停留在本地歌曲页，会让用户误以为点击无效] -> 成功后通过 Activity result 让主壳层直接展开播放器；失败时显式提示并保留当前页。

## Migration Plan

1. 在个人中心页增加“本地歌曲”入口并接通导航。
2. 新增本地歌曲 Repository，接通 `MediaStore` 扫描与本地快照缓存。
3. 新增本地歌曲页面与 ViewModel，先展示缓存，再支持手动重扫。
4. 将本地歌曲列表投影到现有外部队列播放入口，支持播放全部和点击歌曲播放。
5. 补齐本地歌曲页到主壳层的播放成功结果回传，让成功播放后直接展开播放器。
6. 补充入口、缓存快显、重扫和列表播放相关测试与验证。
