## Context

当前缓存链路已经能展示当前播放项的缓存进度，也支持完整缓存命中后优先作为本地缓存源使用。缺口在于：缓存推进通常依赖实际播放读取，下一首或 seek 后的新区间没有独立的提前准备策略。预热需要解决“提前多少”和“何时停止”，否则会退化成整首下载。

## Decisions

- 首版预热只面向在线歌曲，且只预热可解析出稳定资源 key 的目标。
- 预热目标分两类：
  - 当前播放目标的 ahead window：让当前缓存条明显领先播放点。
  - 队列下一首在线歌曲的首段：只在当前播放项缓存完成或无需继续写入缓存后启动，用于降低自动切歌和手动下一首的起播等待。
- 预热预算使用“双阈值先到先停”：
  - 默认预热时长：60 秒。
  - 默认预热字节：8 MiB。
  - 后续设置页可暴露更小/更大的预设，但不得默认整首缓存。
- 预热状态区分 Ready 和 Completed：
  - Ready 表示达到可快速起播阈值，默认阈值为 5 秒或 512 KiB，先到即可发布。
  - Completed 表示达到本轮预算或目标完整缓存，不等同于一定整首缓存完成。
- 预热状态使用独立 `PlaybackPrewarmSnapshot` 投影，字段至少包含目标 ID、目标类型、状态、已缓存字节、目标字节或时长、Ready/Completed 标记和原因文本。
- 同一时间只允许一个后台预热任务，主播放读取和当前播放项缓存优先级高于下一首预热。
- 下一首预热默认被当前项缓存进度门控：当前播放项尚未完整缓存时，调度器不启动下一首预热，避免抢占当前歌曲的网络流量。
- 下一首候选由播放模式决定：顺序/列表循环使用真实下一首，单曲循环不预热下一首，随机模式只在候选稳定后预热。
- 预热必须服从缓存策略：
  - “移动网络只播放不缓存”开启且当前为移动网络时，不写入磁盘预热缓存。
  - “仅 Wi-Fi 自动缓存”开启且当前不是 Wi-Fi 时，不启动自动预热。
  - 完整缓存命中时跳过预热。
- 队列、当前音源、默认音质、播放目标或登录授权上下文变化后，旧预热任务必须取消。
- 预热失败不应改变当前播放状态；仅按缓存失败提示策略记录或提示。

## Data Flow

1. 设置页读取并保存 `PlaybackPrewarmPreferences`。
2. 设置变更通过播放服务自定义命令下发到播放进程。
3. 播放进程在当前项准备成功、队列变更、切到下一首候选时计算预热目标。
4. 预热调度器先评估当前项缓存状态；当前项未完整缓存时，只允许当前项 ahead window 继续推进，不启动下一首预热。
5. 当前项完整缓存或无需继续写入缓存后，预热调度器解析下一首在线 URL 和 resource key，检查缓存命中和策略限制。
6. 预热任务通过缓存核心读取有限 range，将数据写入受管缓存。
7. 预热任务在状态变化时发布 `PlaybackPrewarmSnapshot`。
8. 当前播放项切换到已预热目标时，常规播放准备链路复用已有缓存。

## Error Handling

- 预热 URL 解析失败：不影响当前播放，记录失败原因，等待下一次触发。
- 预热读取失败：取消本次预热；若缓存失败提示开启，可发布“预热缓存失败”类反馈。
- 预热任务被队列或音源变化取消：不展示错误。
- 预热预算耗尽：正常停止并保留已缓存范围。

## Testing

- 设置页测试覆盖预热开关和预算显示/保存。
- 播放服务测试覆盖当前项未完整缓存时延后下一首预热、当前项完整缓存后触发下一首预热、Ready/Completed 状态快照、播放模式候选规则、队列变化取消旧预热、移动网络或仅 Wi-Fi 策略阻止预热。
- 缓存测试覆盖预热只写入有限 range，不会默认拉完整首。
- 回归验证 `:playback-service:testDebugUnitTest`、`:app:testDebugUnitTest`、`:app:assembleDebug`。

## Non-Goals

- 不实现离线下载整首歌曲。
- 不新增后台批量下载队列。
- 不展示队列中非当前项的缓存进度条。
- 不改变当前播放页和 minibar 的缓存进度主语。
- 首版不引入低电量、充电状态或前后台专属预热策略；这些策略后续可单独扩展。
