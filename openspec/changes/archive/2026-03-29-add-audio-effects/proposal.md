## Why

当前播放器已经支持倍速播放，但仍缺少用户可直接感知的默认音效预设，用户无法快速切换到低音增强、人声增强或更清亮的听感模式。现有 native 播放链路已经接入 FFmpeg `avfilter` 并用于 `atempo`，现在补齐统一的音效能力，可以在不额外引入独立 DSP 管线的前提下扩展播放体验。

## What Changes

- 新增播放器音效能力，提供若干可直接选择的默认音效预设，首版以通用听感模式为主，例如原声、低音增强、人声增强、清亮高频、温暖柔和等。
- 将播放页右上角三点按钮收口为“更多操作”半屏浮层入口，并在该浮层首层提供 `音效设置` 与 `倍速设置` 两个入口。
- 让 `音效设置` 以播放器上下文内的独立子页面承载，而 `倍速设置` 继续复用原有离散档位选择，但改为在当前半屏浮层内直接展开。
- 在播放页进度条下方展示当前使用的音效名称，默认音效显示为 `原声`，避免用户切换后缺少持续可见的状态反馈。
- 让音效选择成为当前播放会话状态的一部分，在播放器页、播放服务和 native 播放链路之间保持一致，并在暂停、恢复、Seek、切歌和后台播放时持续生效。
- 基于 FFmpeg 音频滤镜能力实现这些预设效果，复用现有 `avfilter` 接入方式，把音效与倍速收敛到同一条可组合的音频处理链路中。
- 为滤镜不可用、参数非法或链路初始化失败提供安全降级，至少回退到原声音频输出，不破坏现有本地与在线播放能力。

## Capabilities

### New Capabilities
- `playback-audio-effects`: 定义默认音效预设、独立音效设置页、播放会话内状态同步、后台持续生效，以及基于 FFmpeg 滤镜链的安全降级行为。

### Modified Capabilities
- `playback-speed-control`: 补充倍速入口迁移到播放页三点半屏浮层、离散档位在浮层内展开，以及倍速与音效共存时的会话一致性要求。
- `player-expanded-page`: 补充播放页三点更多操作浮层、音效设置独立子页面，以及进度条下方当前音效名称展示的页面行为。

## Impact

- `app` 播放器 UI、ViewModel 与 runtime 需要新增更多操作半屏浮层、音效设置子页面、倍速入口收口、当前音效名称展示和本地状态同步。
- `playback-client` 与 `playback-contract` 需要扩展音效状态同步字段，确保前台快照和共享 metadata 能表达当前音效。
- `playback-service` 需要把音效状态纳入当前播放会话，并保证跨进程控制、后台播放和服务重连时的一致性。
- `player` native C++ 播放链路需要把当前仅处理 `atempo` 的滤镜图扩展为可组合的 FFmpeg 音频效果链。
- 测试需要覆盖三点浮层与音效子页面交互、倍速内联展开、当前音效名称展示、音效状态同步、倍速与音效同时生效、切歌/Seek/后台场景持续生效，以及滤镜失败时的安全降级。
- 不涉及服务端 API 变更；主要新增依赖和风险集中在现有 FFmpeg `avfilter` 能力的正确接入与运行时稳定性。
