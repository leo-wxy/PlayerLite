## ADDED Requirements

### Requirement: 在线解析请求遵循当前启用音源
系统 MUST 让受保护在线播放的音质目录查询与播放地址解析遵循当前启用音源的 baseUrl；当用户未切换音源时，系统 SHALL 继续使用内置默认音源。

#### Scenario: 默认音源下继续走内置 baseUrl
- **WHEN** 用户尚未切换当前音源而系统准备一首具备 `songId` 的在线歌曲
- **THEN** 系统继续使用内置默认音源的 baseUrl 请求 `/song/music/detail`
- **AND** `/song/url/v1` 与 `/song/url` 也继续走该默认 baseUrl

#### Scenario: 自定义当前音源驱动音质目录查询和地址解析
- **WHEN** 用户已将某个自定义 `netease-compatible` 音源设为当前音源且系统准备一首具备 `songId` 的在线歌曲
- **THEN** 系统使用该当前音源的 baseUrl 请求 `/song/music/detail`
- **AND** `/song/url/v1` 与 `/song/url` 也使用该当前音源的 baseUrl

### Requirement: 切换当前音源后旧解析缓存失效
系统 SHALL 在切换当前音源后失效旧音源对应的在线播放 URL 内存缓存与音质目录内存缓存，避免后续在线准备复用旧音源的解析结果。

#### Scenario: 切换当前音源后不再复用旧音源的在线播放 URL 缓存
- **WHEN** 用户把当前音源从 A 切换为 B
- **THEN** 系统清空旧音源下保存的在线播放 URL 内存缓存
- **AND** 后续在线歌曲准备不会继续复用音源 A 的解析结果

#### Scenario: 切换当前音源后不再复用旧音源的音质目录缓存
- **WHEN** 用户把当前音源从 A 切换为 B
- **THEN** 系统清空旧音源下保存的 `/song/music/detail` 音质目录缓存
- **AND** 后续当前歌曲的音质目录查询重新走音源 B

### Requirement: 切换当前音源时当前在线歌曲重新准备
系统 MUST 在切换当前音源后，对当前会话中的在线歌曲重新准备，以保证实际播放链接、音质目录和当前音源保持一致；对本地文件与 `content://` 音源则 MUST 只更新配置而不强制重准备。

#### Scenario: 正在播放在线歌曲时切换音源会重新准备并继续播放
- **WHEN** 用户在一首具备 `songId` 的在线歌曲播放过程中切换当前音源
- **THEN** 系统清空旧音源下的解析缓存并重新准备该歌曲
- **AND** 如果切换前处于播放态，则重新准备后继续播放

#### Scenario: 已暂停的在线歌曲切换音源后保持暂停位置
- **WHEN** 用户在一首已暂停的在线歌曲上切换当前音源
- **THEN** 系统重新准备该歌曲
- **AND** 重新准备后仍保持暂停态与当前位置

#### Scenario: 本地歌曲切换音源时不触发在线重准备
- **WHEN** 当前曲目是本地文件或 `content://` 音源且用户切换当前音源
- **THEN** 系统更新当前音源配置
- **AND** 不会因为这次切换把本地歌曲错误地送入在线播放重准备链路
