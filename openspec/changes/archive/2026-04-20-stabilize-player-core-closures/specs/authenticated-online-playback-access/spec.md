## ADDED Requirements

### Requirement: 在线资源失效使用统一准备结果分类
系统 SHALL 将 URL 过期、版权不可用、歌曲下架、无可播直链、试听片段和短时网络失败映射为可解释的在线播放准备结果，而不是把所有失败都视为通用网络错误或登录失效。

#### Scenario: URL 过期触发重新解析
- **WHEN** 当前在线歌曲已有的播放地址过期或服务端返回等价的过期结果
- **THEN** 系统重新请求在线地址解析
- **AND** 成功获得新地址后继续准备同一首歌曲

#### Scenario: 无版权或下架不误判为登录失效
- **WHEN** 在线地址解析或可用性检查确认当前歌曲无版权、下架或无完整可播音源
- **THEN** 系统以明确资源不可用结果结束当前准备
- **AND** 系统不把该失败误判为需要重新登录

#### Scenario: 短时网络失败保留可重试语义
- **WHEN** 在线地址解析或 range 读取因为短时网络异常失败
- **THEN** 系统返回可重试准备结果
- **AND** 播放服务可以继续对同一播放目标执行有限重试

### Requirement: 在线资源兜底限制在当前单曲范围
系统 SHALL 只针对当前准备的单曲执行资源刷新、备用 source 解析或失败兜底；这些兜底结果 MUST 不自动改写整条歌单、收藏关系或全局默认音源。

#### Scenario: 当前单曲进入备用 source 兜底
- **WHEN** 当前 source 无法为目标歌曲返回可用完整音源
- **AND** 系统存在可用备用 source
- **THEN** 系统可以为当前单曲尝试备用 source 解析
- **AND** 本次兜底不改变全局当前 source

#### Scenario: 所有兜底失败后保持队列上下文
- **WHEN** 当前单曲的主 source 和备用 source 都无法返回可用播放资源
- **THEN** 系统以当前歌曲资源不可用结束本次准备
- **AND** 当前播放队列仍保留该歌曲条目，供用户重试、切歌或查看详情
