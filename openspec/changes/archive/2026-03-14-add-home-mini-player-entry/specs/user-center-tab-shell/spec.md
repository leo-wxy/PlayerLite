## MODIFIED Requirements

### Requirement: 首页概览态与播放展开态属于同一 Home 域
系统 MUST 将首页概览态建模为承载发现内容、顶部悬浮搜索框与底部迷你播放条的 Home Tab 概览页，并与播放展开态保持在同一 Home Tab 域内，而不是完全脱离首页上下文的独立一级页面或无关全局路由；同时，首页概览态底部迷你播放条 SHALL 以接近音乐 App minibar 的胶囊式结构承载当前播放上下文，优先展示左侧封面、中间歌曲信息与右侧组合控件区，而不是继续作为独立的通用操作卡片存在。

#### Scenario: 首页概览态展示发现内容首页
- **WHEN** 用户位于首页概览态
- **THEN** 系统展示首页发现内容与顶部悬浮搜索框
- **AND** 当前选中的一级 Tab 仍保持为首页

#### Scenario: 首页底部迷你播放条主体触发播放展开态
- **WHEN** 用户位于首页概览态并点击底部迷你播放条的主体区域
- **THEN** 系统进入 Home 域内的播放展开态
- **AND** 当前选中的一级 Tab 仍保持为首页

#### Scenario: 首页底部迷你播放条歌曲展示区左滑切到下一首
- **WHEN** 用户位于首页概览态并在底部迷你播放条中间歌曲展示区域向左横滑，且当前播放队列存在下一首
- **THEN** 系统切换到当前队列中的下一首
- **AND** 不会因此直接进入播放展开态

#### Scenario: 首页底部迷你播放条歌曲展示区右滑切到上一首
- **WHEN** 用户位于首页概览态并在底部迷你播放条中间歌曲展示区域向右横滑，且当前播放队列存在上一首
- **THEN** 系统切换到当前队列中的上一首
- **AND** 不会因此直接进入播放展开态

#### Scenario: 首页底部迷你播放条歌曲展示区横滑未满足切歌条件时回弹
- **WHEN** 用户位于首页概览态并在底部迷你播放条中间歌曲展示区域横滑，但当前没有可切换的上一首/下一首，或横滑距离未达到切歌阈值
- **THEN** 系统保持当前播放项不变
- **AND** 迷你播放条歌曲展示区域回到稳定位置

#### Scenario: 首页底部迷你播放条展示当前播放信息
- **WHEN** 用户位于首页概览态查看底部迷你播放条
- **THEN** 系统以胶囊式容器展示该入口
- **AND** 左侧展示封面承载位
- **AND** 中间按“歌曲名在上、歌手信息在下”的顺序展示当前播放信息
- **AND** 右侧以同一组控件承载播放进度、播放状态与播放列表入口
- **AND** 歌曲名、歌手与封面优先来自当前播放上下文的运行时字段，而不是仅靠文件名解析猜测

#### Scenario: 首页底部迷你播放条长歌名以跑马灯承载
- **WHEN** 用户位于首页概览态查看底部迷你播放条，且当前歌曲名长度超过中间歌曲展示区域的可用宽度
- **THEN** 系统以单行跑马灯方式展示歌曲标题
- **AND** 迷你播放条整体高度、右侧组合控件区与歌手信息排布保持稳定

#### Scenario: 首页底部迷你播放条与首页视觉保持统一
- **WHEN** 用户位于首页概览态查看页面底部迷你播放条
- **THEN** 系统以与首页搜索框和发现内容一致的视觉语言展示该入口
- **AND** 迷你播放条整体观感更接近音乐 App 的 minibar，而不是通用按钮卡片

#### Scenario: 首页迷你播放条右侧播放状态按钮独立响应
- **WHEN** 用户点击首页迷你播放条右侧的播放状态按钮
- **THEN** 系统仅执行播放或暂停动作
- **AND** 不会因此直接进入播放展开态

#### Scenario: 首页迷你播放条右侧列表按钮打开播放列表
- **WHEN** 用户点击首页迷你播放条右侧的播放列表入口
- **THEN** 系统打开当前播放列表面板
- **AND** 不会因此直接进入播放展开态

#### Scenario: 首页迷你播放条缺少真实封面时保持稳定布局
- **WHEN** 当前播放项缺少可直接展示的真实封面资源
- **THEN** 系统在迷你播放条左侧展示固定尺寸的占位封面承载位
- **AND** 迷你播放条的整体高度、信息排布与点击区域保持稳定

#### Scenario: 首页与播放页保留共享视觉锚点
- **WHEN** 用户从首页概览态进入播放展开态
- **THEN** 系统保持首页与播放展开态属于同一 Home 上下文
- **AND** 为歌曲封面、背景或卡片等共享元素预留联动过渡锚点

#### Scenario: 首页到播放展开态的过渡保持单段连续
- **WHEN** 用户从首页概览态进入播放展开态
- **THEN** 系统使用 Home 域外层过渡承载主要的页面展开运动
- **AND** 不以多层重复整页 reveal 造成明显的二段式切换僵硬感
- **AND** Home 内容区不会因为底部 Tab 退场或首页发现内容重排而产生明显抖动

#### Scenario: 播放展开态隐藏底部 Tab 并提供返回入口
- **WHEN** 用户进入 Home 域内的播放展开态
- **THEN** 系统隐藏底部 Tab
- **AND** 系统提供显式返回按钮用于回到首页概览态

#### Scenario: 返回首页时恢复发现概览上下文
- **WHEN** 用户从播放展开态返回首页概览态
- **THEN** 系统恢复此前的首页发现内容上下文
- **AND** 不将用户带离当前 Home Tab 域
