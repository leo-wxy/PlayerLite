## Why

当前搜索能力已经能够产出歌手、歌单、专辑等不同结果类型，并为结果项保留详情目标，但点击后的实际跳转与详情承载还没有完整实现，导致搜索链路停留在“能看到结果、不能继续进入内容”的半成品状态。现在需要补齐歌手详情页、歌单详情页、专辑详情页与对应接口接入，让搜索能力真正闭环，并把详情页视觉统一到更偏云音乐的音乐内容体验。

## What Changes

- 新增歌手详情页，接入“获取歌手详情”“歌手简要百科信息”与“歌手热门 50 首歌曲”接口，承载歌手基础信息、百科摘要/全文、热门歌曲列表、头部展示与详情加载/错误/空态。
- 新增歌单详情页，接入“获取歌单详情”与“获取歌单所有歌曲”接口，承载歌单头部信息、歌曲列表与对应状态页。
- 新增歌单详情页，接入“获取歌单详情”与“获取歌单所有歌曲”接口，并把歌曲列表改成分页加载，避免长歌单一次性拉取和渲染卡住首屏。
- 新增专辑详情页，接入“获取专辑内容”与“专辑动态信息”接口，并把歌曲列表改成分页加载，避免长专辑曲目一次性拉取和渲染卡住首屏。
- 修改搜索结果交互，让歌手结果、歌单结果与专辑结果点击后进入对应详情页，而不是继续停留在无跳转或占位行为。
- 修改首页发现流交互，让首页 banner、推荐卡与快捷入口都具备明确点击反馈，并在可识别为歌手、歌单、专辑时进入站内详情；无法站内承载但可解析的链接交给宿主打开，仍无法处理的目标给出明确兜底反馈。
- 修改个人中心交互，让收藏歌手、用户歌单和收藏专栏列表项都具备明确点击反馈，其中歌手与歌单进入对应详情页，当前暂未承载的专栏内容提供稳定提示而不是静默无响应。
- 统一歌手详情页、歌单详情页与专辑详情页的视觉方向，使头部氛围、封面/头像、标题信息层级与列表排布更偏云音乐风格，而不是通用列表页样式。
- 将歌单详情页与专辑详情页头部也升级为“基于封面主色提取的渐变氛围层 + 失败兜底默认渐变”，让三类详情页的头部质感保持一致。
- 为歌手头像点击放大预览、歌手/歌单/专辑简介点击浮层展开补充交互承载，并让全文浮层支持滑动查看，让详情信息不被摘要截断。
- 调整搜索结果页的类型筛选 Tab，使不同结果类型切换时保持固定高度，避免因文案和状态变化导致筛选条抖动。
- 将歌手详情页头部升级为“基于头像主色提取的渐变氛围层 + 失败兜底默认渐变”，进一步贴近云音乐的视觉体验。
- 为详情页补充必要的加载态、失败态与重试入口，保证接口返回慢或失败时页面仍可用。

## Capabilities

### New Capabilities
- `artist-detail-page`: 定义歌手详情页的进入方式、基础信息承载、头部展示与状态页要求。
- `playlist-detail-page`: 定义歌单详情页的进入方式、歌单头部信息、完整歌曲列表承载与状态页要求。
- `album-detail-page`: 定义专辑详情页的进入方式、专辑内容与动态信息承载、歌曲列表与状态页要求。

### Modified Capabilities
- `search-page`: 补充搜索结果点击契约，要求歌手结果、歌单结果与专辑结果能够进入对应详情页，而不是仅保留本地 route target。
- `search-page`: 补充结果类型筛选 Tab 的固定高度契约，要求搜索结果页切换不同结果类型时筛选条保持稳定高度。
- `homepage-discovery-content`: 补充首页发现内容 item 的点击契约，要求所有首页内容入口都具备明确反馈，并在可识别时进入站内详情或外部链接。
- `user-center-tab-shell`: 补充个人中心内容 item 的点击契约，要求收藏歌手、用户歌单和收藏专栏列表项都具备明确反馈与可预期结果。

## Impact

- 影响搜索结果点击后的路由接线，重点包括 `feature-search` 中的 `SearchRouteTarget` / `SearchRouteHandler` 与 `app` 侧宿主跳转实现。
- 需要在 `app` 中新增歌手详情页、歌单详情页、专辑详情页相关的 Activity / Compose Screen / ViewModel / state / repository 装配。
- 需要扩展首页发现流与个人中心列表的数据模型、mapper 与宿主层点击处理，让它们复用现有 artist / playlist / album 详情路由与站外链接处理。
- 需要补充“获取歌手详情”“歌手简要百科信息”“歌手热门 50 首歌曲”“获取歌单详情”“获取歌单所有歌曲”“获取专辑内容”“专辑动态信息”七类远端数据访问、DTO 映射与页面状态建模，并继续复用现有 `JsonHttpClient` 与登录态透传约定。
- 需要为歌单与专辑详情的歌曲列表补充分页加载状态、续页加载与失败重试承载，避免长列表一次性加载拖慢详情页可见性。
- 需要新增 `artist-detail-page`、`playlist-detail-page` 与 `album-detail-page` spec，并为 `search-page`、`homepage-discovery-content`、`user-center-tab-shell` 增补 delta spec，以约束搜索、首页、个人中心与详情页之间的交互闭环。
