## MODIFIED Requirements

### Requirement: 启动时恢复并校验持久化数据
系统 SHALL 在启动阶段按用户的“启动后恢复上次播放”偏好决定是否尝试恢复播放列表状态，并在恢复时对数据结构、条目有效性与激活索引进行校验后再应用。

#### Scenario: 启动恢复开启且持久化数据有效时恢复
- **WHEN** 启动后恢复上次播放已开启
- **AND** 启动时读取到结构完整且条目有效的持久化数据
- **THEN** 系统恢复播放列表顺序与激活项，使用户可继续上次状态

#### Scenario: 启动恢复关闭时不应用历史播放上下文
- **WHEN** 启动后恢复上次播放已关闭
- **THEN** 系统不应用上次播放列表、激活项和播放位置
- **AND** 系统仍保留后续新播放会话的持久化能力

#### Scenario: 存在部分无效条目时恢复
- **WHEN** 启动时读取到部分条目无效或不可用的持久化数据
- **THEN** 系统跳过无效条目，保留有效条目，并修正激活索引到有效范围

## ADDED Requirements

### Requirement: 播放会话位置恢复受断点续播开关控制
系统 SHALL 在启动恢复播放上下文时按用户的“断点续播”偏好决定是否恢复上次播放位置；关闭断点续播时可恢复队列和当前项，但播放位置 MUST 从 0 开始。

#### Scenario: 断点续播开启时恢复上次位置
- **WHEN** 启动后恢复上次播放已开启
- **AND** 断点续播已开启
- **AND** 系统读取到有效播放位置
- **THEN** 系统恢复当前项并展示上次播放位置

#### Scenario: 断点续播关闭时从头开始
- **WHEN** 启动后恢复上次播放已开启
- **AND** 断点续播已关闭
- **AND** 系统读取到有效播放位置
- **THEN** 系统恢复当前项
- **AND** 播放位置从 0 开始
