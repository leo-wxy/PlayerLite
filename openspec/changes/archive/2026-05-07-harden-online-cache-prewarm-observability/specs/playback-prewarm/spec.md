## MODIFIED Requirements

### Requirement: 预热任务可取消且不污染新播放上下文
系统 SHALL 在播放上下文变化后取消旧预热任务，避免旧队列、旧音源或旧音质下的缓存任务继续占用资源。系统 MUST 丢弃与当前播放上下文签名不匹配的预热状态发布、完成回调和失败回调。

#### Scenario: 队列变化取消旧预热
- **WHEN** 预热任务正在处理旧队列中的下一首歌曲
- **AND** 用户切换播放队列或移除该目标
- **THEN** 系统取消旧预热任务
- **AND** 旧目标不得继续作为当前队列的预热结果展示或使用

#### Scenario: 音源或音质变化取消旧预热
- **WHEN** 预热任务依赖当前音源或默认音质解析出的在线播放资源
- **AND** 用户切换音源或默认音质
- **THEN** 系统取消旧预热任务
- **AND** 后续预热使用新的音源和音质上下文重新计算 resource key

#### Scenario: 旧预热任务晚到结果被丢弃
- **WHEN** 旧预热任务已经被新的播放上下文替换
- **AND** 旧任务的 Ready、Completed、Failed 或 Canceled 回调晚于新任务到达
- **THEN** 系统丢弃该旧回调
- **AND** 当前预热快照仍保持新播放上下文对应的目标和状态

#### Scenario: 切换音质后旧音质预热完成不覆盖新音质快照
- **WHEN** 默认音质从旧音质切换为新音质
- **AND** 新音质预热任务已经开始
- **AND** 旧音质预热任务随后完成
- **THEN** 系统不得用旧音质的完成状态覆盖新音质预热快照
- **AND** 后续预热和播放准备只复用新音质上下文下的 resource key
