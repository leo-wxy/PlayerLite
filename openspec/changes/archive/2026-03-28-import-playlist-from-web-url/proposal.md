## Why

当前应用已经具备在线搜索、歌单详情和播放列表管理能力，但还不能把用户在外部平台分享出来的歌单网页链接直接转成应用内可播放的歌单。先支持网易云歌单和 QQ 音乐歌单网页链接导入，可以显著降低用户手工搜歌重建歌单的成本，也为后续扩展更多外部来源建立统一入口。

## What Changes

- 新增“从网页链接导入歌单”能力，首批支持网易云歌单和 QQ 音乐歌单网页链接。
- 系统识别链接来源与关键信息，解析歌单标题、来源信息和可用于匹配曲目的关键字或曲目线索，并据此生成应用内歌单。
- 导入流程对成功匹配、部分匹配和无法匹配的曲目给出可解释结果，而不是静默失败。
- 导入成功后，生成的歌单可以进入现有应用内歌单/播放列表链路，供用户浏览、播放或替换当前队列。

## Capabilities

### New Capabilities
- `web-playlist-import`: 定义支持平台网页链接的识别、解析、导入结果反馈，以及将外部歌单线索生成应用内歌单的行为。

### Modified Capabilities
- `playlist-management`: 扩展播放列表管理规格，使导入生成的歌单能够作为合法的队列来源接入现有播放列表替换、激活项选择和后续播放流程。

## Impact

- Affected code: `app` 侧新增歌单导入入口、链接解析/导入 repository、导入结果到 `PlaylistController`/`PlayerRuntime` 的接入，以及相关 UI 状态与错误提示。
- APIs / dependencies: 依赖当前在线能力链路与仓库内配置的 API base URL `http://139.9.223.233:3000`；可能需要新增或扩展用于解析外部歌单链接、按关键字匹配曲目或拉取歌单元数据的接口能力。
- Systems: 需要覆盖支持来源识别、导入失败降级、部分匹配可见性，以及导入后与现有播放列表持久化/播放流程的一致性。
