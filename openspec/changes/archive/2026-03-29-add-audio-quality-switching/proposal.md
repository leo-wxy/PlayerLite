## Why

当前在线歌曲详情链路只调用 `/song/detail`，`MusicInfo` 里也只保留标题、歌手、专辑和时长，播放器无法知道一首歌实际有哪些可用音质，更无法提供和歌曲真实可用档位一致的切换能力。现在需要接入 `/song/music/detail` 补齐各音质文件信息，把高清环绕声、沉浸环绕声、超清母带等档位纳入播放准备依据，避免继续依赖固定默认码率请求。

## What Changes

- 新增歌曲音质信息能力，使用 `/song/music/detail?id=<songId>` 获取当前歌曲全部可用音质及其文件信息，而不再只依赖 `/song/detail` 的基础元数据。
- 为在线歌曲建立统一的音质元数据结构，至少覆盖音质标识以及接口返回的 `br`、`size`、`vd`、`sr` 等字段，支持区分普通、高品质、无损、高清环绕声、沉浸环绕声、超清母带等可选档位。
- 在播放器上下文中提供基于“当前歌曲实际可用音质”的展示与切换能力，避免向用户暴露当前歌曲并不支持的档位。
- 在播放页进度条下方使用一行联合状态区承载音质 / 音效状态：默认只显示当前实际生效的音质；只有当前存在非默认音效时，才追加显示 `音质 · 音效`。
- 让联合状态区中的 `音质` 片段直接弹出音质选择浮层；当当前音源不支持音质切换时，不伪造音质项，联合状态区只承担真实生效值展示。
- 在线播放准备链路按当前选中的音质重新解析播放地址和缓存身份，不再默认回落到固定 bitrate；当目标音质不可用或解析失败时，系统按兼容策略回退到可用档位并保持播放可继续。
- 为音质详情获取、音质选中状态同步、在线播放重新准备与回退路径补充测试。

## Capabilities

### New Capabilities
- `playback-audio-quality-switching`: 定义歌曲音质详情获取、当前歌曲可用音质展示与切换、音质元数据表达，以及按所选音质准备在线播放的行为。

### Modified Capabilities
- `player-expanded-page`: 补充进度条下方音质 / 音效联合状态区、音质片段直达音质浮层，以及音质不支持时的退化展示规则。
- `authenticated-online-playback-access`: 补充在线地址解析、缓存键与播放准备对所选音质 / level 的约束，避免继续固定使用默认 bitrate 或忽略歌曲实际可用档位。

## Impact

- `app` 侧需要扩展歌曲详情获取与映射链路，补齐 `/song/music/detail` 的接入、音质元数据模型，以及播放器页的音质展示与切换入口。
- `playback-contract` 与前后台共享播放状态需要新增音质相关字段，确保前台界面、后台播放服务和共享 metadata 能表达当前歌曲的可用音质、当前偏好与当前实际生效值。
- `playback-service` 需要调整 `OnlinePlaybackSupport` 一类的在线播放解析与缓存逻辑，让所选音质参与 URL 解析、缓存身份和失败回退判断。
- 测试需要覆盖音质详情映射、缺失档位过滤、当前歌曲音质切换、进度条下方联合状态区展示、音质片段点击、所选音质驱动的在线播放准备，以及目标音质不可用时的兼容回退。
- 不涉及服务端 API 变更，主要依赖仓库当前已配置的 API Base URL 下现有 `/song/music/detail`、`/song/url/v1` 与相关在线播放接口协同工作。
