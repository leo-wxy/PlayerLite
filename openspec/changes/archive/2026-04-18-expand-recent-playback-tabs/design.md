## Overview

最近播放页升级为一个多类型总页，统一承接本机、歌曲、视频、声音、歌单、专辑和播客 7 类最近播放内容。页面延续当前用户中心入口，但内部不再绑定“歌曲”语义，而是通过顶部标签切换不同内容源。

首版目标是把 6 个远端接口和 1 个本机最近听歌缓存来源真实渲染到 UI 上，并保持页面结构、列表节奏和状态表达一致；不在这条变更里继续扩展视频、声音、播客等类型的详情或播放承接能力。

## Goals

- 用户中心“最近播放”入口打开的是多类型最近播放总页，而不是单一歌曲页
- 本机 tab 固定排在最前，并作为默认优先展示 tab
- 6 个远端最近播放接口分别独立请求、独立渲染
- 本机最近听歌缓存来自本机真实播放行为沉淀，不依赖服务端接口
- 默认进入本地 tab，其它 tab 按需加载，不做首屏并发全拉
- 列表样式与搜索结果页保持同一套骨架语言
- 新增 tab 首版只做渲染，不因为缺少详情页承接而伪造错误跳转

## Non-Goals

- 不新增视频、声音、播客的详情页或专用播放页
- 不改用户中心其它入口结构
- 不回退当前歌曲 tab 已有的点击和更多能力

- 顶部 `TopAppBar` 标题仍为“最近播放”
- 标题下方新增水平标签栏：
  - 本机
  - 歌曲
  - 视频
  - 声音
  - 歌单
  - 专辑
  - 播客

### List Style

- 整体参考搜索结果页：
  - 左侧封面
  - 中间标题 / 副标题 / 辅助信息
  - 右侧仅在歌曲 tab 保留已有更多入口
- 不同类型只调整：
  - 封面圆角和比例
  - 副标题与 meta 字段
  - 是否显示右侧次级入口

### Interaction

- 本机 / 歌曲 tab：保留当前已实现播放承接交互
- 视频 / 声音 / 歌单 / 专辑 / 播客 tab：
  - 首版只渲染，不新增点击功能
  - 不显示三点更多

## Data Model

### Tabs

- 新增 `RecentPlaybackTab` 枚举，作为页面状态和远端请求的唯一来源
- 每个 tab 独立维护：
  - `Idle`
  - `Loading`
  - `Empty`
  - `Error`
  - `Content`
- 每个 tab 的已加载结果缓存保留在 `ViewModel` 内存状态中
- 本地 tab 使用独立缓存，不因切换到远端 tab 而重建
- 用户在标签之间来回切换时，不清空已加载内容
- 只有显式点击刷新，或首次进入未加载 tab 时，才重新发起请求
- 每个 tab 最多保留最近 100 条记录；本地 tab 超过上限时按最近播放时间裁剪旧记录

### Items

- 本机 / 歌曲继续使用现有独立歌曲模型
- 新增通用 `RecentPlaybackListItemUiModel`，用于：
  - 视频
  - 声音
  - 歌单
  - 专辑
  - 播客

该模型包含：
- `id`
- `title`
- `subtitle`
- `imageUrl`
- `meta`
- `badge`

## Data Fetching

数据来源分为两类：

- 本机 tab：
  - 来自本机播放行为写入的最近听歌缓存
  - 不依赖服务端上报、同步或补全接口
  - 读取时按最近播放时间倒序返回，最多 100 条
- 远端 tab：
  - 由 `UserCenterRemoteDataSource` 提供对应读取接口

`UserCenterRemoteDataSource` 新增 5 个读取接口：

- `/record/recent/video`
- `/record/recent/voice`
- `/record/recent/playlist`
- `/record/recent/album`
- `/record/recent/dj`

规则：
- 本机 tab 作为默认首屏数据源，进入页面时优先读取
- 远端 tab 每个请求默认 `limit=100`
- 本地 tab 与远端 tab 都只在首次进入该 tab 时加载
- 切回已加载过的 tab 时优先复用缓存结果
- 刷新按钮触发当前 tab 重新读取或重新拉取

## Local Playback Source

- 本机最近听歌缓存记录真实发生在本机的歌曲播放行为，不区分在线歌曲还是本地文件
- 记录范围以当前歌曲播放能力为准，不扩大到视频、播客等尚未接入的非歌曲来源
- 当同一首歌曲重复播放时，以最近一次播放时间更新排序，不重复膨胀列表
- 本机记录持久化细节复用现有本地存储能力，不在该变更里引入服务端依赖

## Tab Ordering And Default

- 标签顺序固定为：本机、歌曲、视频、声音、歌单、专辑、播客
- 首次进入页面默认选中本机 tab
- 如果本机 tab 为空，仍保持本机 tab 为默认选中，只展示对应空态，不自动跳到歌曲 tab

## Cache Strategy

- 页面级状态需要缓存每个 tab 最近一次成功加载的结果和状态
- 本机 tab 切入切出时复用缓存，避免重复读取本机记录
- 远端 tab 切入切出时复用缓存，避免重复请求接口
- 显式刷新只影响当前 tab 的缓存内容，不清空其它 tab

## Rendering Rules By Tab

### 本机

- 主标题：歌曲名
- 副标题：歌手
- 辅助信息：专辑名或本地来源标识
- 主点击直接使用当前本机最近听歌列表替换播放队列并开始播放
- 保留当前三点更多能力

### 歌曲

- 主标题：歌曲名
- 副标题：歌手
- 辅助信息：专辑名
- 主点击直接使用当前歌曲最近播放列表替换播放队列并开始播放
- 保留当前三点更多能力

### 视频

- 主标题：视频标题
- 副标题：创作者 / 艺人
- 辅助信息：播放量或类型信息
- 仅渲染，不点击

### 声音

- 主标题：内容标题
- 副标题：主播 / 作者
- 辅助信息：分类或节目期数
- 仅渲染，不点击

### 歌单

- 主标题：歌单名
- 副标题：创建者
- 辅助信息：曲目数
- 首版仅渲染，不点击

### 专辑

- 主标题：专辑名
- 副标题：歌手
- 辅助信息：发行时间 / 曲目数
- 首版仅渲染，不点击

### 播客

- 主标题：播客名
- 副标题：DJ / 主播
- 辅助信息：节目数或分类
- 仅渲染，不点击
