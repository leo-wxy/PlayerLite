## MODIFIED Requirements

### Requirement: 歌单详情页 detail chrome 参考专辑/歌手详情页
系统 SHALL 让歌单详情页的 detail chrome 与专辑/歌手详情页保持一致的视觉语言，包括沉浸式 hero（绘制至状态栏下方）、底部 scrim 保证文字可读性、滚动折叠时 top bar 渐显标题与渐变背景，以及更贴近专辑详情页的歌曲条目行样式。

#### Scenario: 歌单详情页展示沉浸式头部与歌曲列表
- **WHEN** 用户查看歌单详情页
- **THEN** 系统在页面顶部展示沉浸式歌单详情头部（hero 绘制至状态栏下方）
- **AND** 系统在 hero 底部提供稳定的 scrim/遮罩以保证标题与按钮在不同封面下可读
- **AND** 返回入口固定在详情页外层且不随正文滚动（通过 overlay top bar 承载）
- **AND** 系统在用户滚动页面时渐显顶部折叠栏标题，并以渐变背景过渡到页面背景色
- **AND** 当封面缺失时系统仍提供稳定的占位主视觉（首字母与主题色占位）
- **AND** 歌单作者与“XX 首歌曲”在头部信息中以可区分的层级或视觉块承载
- **AND** “播放全部”入口在头部信息区域可直接触达
- **AND** 系统通过 tab 导航展示“歌曲”和“简介”两块内容，且用户可在两者之间切换
- **AND** 歌曲列表在“歌曲”tab 中展示，并以更接近专辑详情页的紧凑卡片行样式承载（圆角 card 行）
- **AND** 歌单简介在“简介”tab 中展示，并支持滚动查看全文

### Requirement: 歌单详情页独立承载动态信息并在真实播放后更新播放量
系统 SHALL 为歌单详情页独立加载评论数、收藏状态与播放数等动态信息，并在真实播放已经成功开始后回写播放量。

#### Scenario: 动态信息加载成功时补齐歌单统计信息
- **WHEN** 歌单详情页成功请求当前歌单的动态信息
- **THEN** 系统在详情页展示该歌单的评论数、收藏状态与动态播放数
- **AND** 这些动态信息与已有头部信息、歌曲列表共同组成完整详情页
- **AND** 动态信息区域采用与专辑详情页一致的 card 统计区承载（圆角、surface 背景、三列指标），避免以独立大块卡片打断页面节奏

#### Scenario: 动态信息失败不阻断主体内容
- **WHEN** 歌单详情页动态信息请求失败
- **THEN** 已显示的歌单头部和歌曲列表继续保留
- **AND** 动态信息区域仅展示局部失败提示或重试入口
- **AND** 页面主结构保持稳定，不会因为动态信息失败回退为整页初始加载或白屏失败态

#### Scenario: 真实播放开始后更新歌单播放量
- **WHEN** 用户从歌单详情页成功发起播放
- **THEN** 系统在播放真正开始后更新当前歌单播放量
- **AND** 不会在用户仅进入详情页时就更新播放量
