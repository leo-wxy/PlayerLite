## Context

播放页最近的痛点不是“少一个按钮”，而是整体视觉层次和交互心智不稳定：外层壳层和页面内标题重复叠顶栏，按钮与状态栏互相打架，封面和底部信息不断抢空间，buffering 提示又会把底部内容整体挤动。用户已经把方向收得很明确：顶部是正方形封面，底部是稳定的信息区和播控区，背景取封面颜色，页面保持沉浸式。

## Decisions

1. 顶栏收成一层。
   - `PlayerScreen` 自己承载返回、标题“歌曲”和分享按钮。
   - 外层 `PlayerExpandedScreen` 在播放器展开态不再叠第二层顶栏。

2. 主视觉采用“顶部正方形封面 + 深色基底 + 轻模糊封面铺底 + 深色遮罩”的混合背景。
   - 背景复用当前封面已经加载出来的位图结果取色，并叠加同源封面的轻模糊铺底，不再额外发起第二次独立图片请求。
   - 模糊铺底必须仍能辨认出封面轮廓，不能把背景收成只剩主色变化的纯色感。
   - 封面请求优先拿可读软件位图；如果结果仍然不可直接采样，则先复制为可读位图，失败时保留上一帧背景色并安全回退。
   - 取色结果仍保持深色沉浸式氛围，但亮色封面不能再把背景压成一片发黑的纯黑感。
   - 封面保持正方形并放在顶部内容层，不再做唱片式圆盘主视觉。

3. 底部信息区与播控区固定分层。
   - 歌名、歌手和“歌词待补充”固定在底部信息区。
   - 当当前曲目拥有稳定 `artistId` 时，歌手名直接作为可点击入口跳转到歌手详情页；没有 `artistId` 时仍按静态文本展示。
   - 进度条改成更接近播放器 progressbar 的细轨道 + 小圆点样式，避免继续使用默认表单式 Slider 视觉。
   - 进度条与播控区整体更靠下，左右外边距与进度条对齐。
   - 四个侧边按钮增大图标和点击区，中间主按钮仍明显大于四个侧边按钮，但比例要收在更协调的区间，避免主按钮压过整排播控。

4. buffering 提示锚在封面右下角，且不阻断关键播控。
   - seek / buffering 时显示明确状态，但不再把整排播控统一禁用成灰态。
   - 播放模式、上一首、下一首和播放列表仍保持可操作。

## Verification

- 覆盖真实顶栏基线对齐、正方形封面、混合背景层、底部控件位置、侧边按钮尺寸与小屏不重叠
- 覆盖 `HARDWARE bitmap` / 不可读位图不会让播放页崩溃，而是安全回退
- 覆盖 buffering 提示显示且播控仍可点击
- 覆盖歌手名在存在稳定 `artistId` 时可直接跳转歌手详情页
- 覆盖 MediaSession 点击直接进入播放器展开页
