## Why

当前设置页已经承载账户、播放偏好、缓存和音源管理，但页面结构仍偏“表单堆叠”：关键状态分散在多个卡片内部，用户需要上下滚动才能确认当前默认音质、缓存占用、缓存上限和当前音源。后续在线缓存、音源和播放偏好能力继续增加时，这种结构会让设置页越来越难扫读。

## What Changes

- 重构设置页信息架构，改成账户入口 + 分组状态行，展示登录态、默认音质、缓存占用和当前音源。
- 调整设置页分组顺序和文案层级，让“播放与缓存”作为核心分组先于详细缓存操作出现。
- 优化音源管理区，把导入入口、当前音源和音源列表状态拆清楚，减少长表单带来的视觉噪音。
- 保持现有业务能力不变：登录/退出、默认音质、缓存上限、缓存刷新/清理、在线/本地导入音源、设为当前音源、删除音源都继续可用。
- 补充 Compose UI 定向测试，覆盖分组顺序、关键状态行、关键操作回调和音源状态展示。

## Impact

- 影响范围：`SettingsActivity` 中的 Compose UI 结构、设置页 UI 测试、OpenSpec 设置页相关 specs。
- 不改动：播放服务、缓存清理协议、音源导入/持久化模型、默认音质保存链路。
- 风险：测试依赖的 testTag 可能随 UI 重组变化，需要同步更新并保留既有关键 tag 的兼容性。
