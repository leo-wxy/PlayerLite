## ADDED Requirements

### Requirement: 搜索页空查询态采用独立返回键与轻量搜索栏
系统 SHALL 在搜索页空查询态下，以左侧独立返回键和右侧轻量圆角搜索栏承载返回与输入能力，而不是继续让返回动作与搜索容器混成同一视觉层。

#### Scenario: 顶部搜索头将返回键与搜索框分层展示
- **WHEN** 用户进入搜索页且当前处于空查询态
- **THEN** 顶部左侧展示独立返回按钮
- **AND** 右侧展示包含搜索 icon 与占位文案的轻量圆角搜索框
- **AND** 返回按钮与搜索框之间保持稳定间距和共同对齐基线

### Requirement: 搜索页空查询态采用标签化历史区与榜单式热门搜索区
系统 SHALL 在搜索页空查询态下，将历史搜索展示为轻量标签，并将热门搜索展示为带统一容器的榜单式列表，而不是继续使用松散的默认列表拼装首屏内容。

#### Scenario: 历史搜索展示为可清除的轻量标签
- **WHEN** 用户查看存在历史记录的搜索页空查询态
- **THEN** 历史搜索以轻量标签样式展示
- **AND** 每个标签在同一视觉单元中承载历史标识、关键词与单项移除入口
- **AND** 区域级清空操作保持为独立主强调入口

#### Scenario: 热门搜索展示为统一榜单容器
- **WHEN** 用户查看空查询态下的热门搜索区域
- **THEN** 系统以统一的浅底容器承载热门搜索列表
- **AND** 每一行保持一致的高度、左右留白与分隔节奏
- **AND** 热门搜索入口不再表现为厚重的独立卡片堆叠

### Requirement: 搜索结果页采用固定搜索头与紧凑音乐结果列表
系统 SHALL 让搜索结果页以固定搜索头、固定类型切换栏和紧凑音乐结果列表承载搜索结果，而不是继续让结果页在结构与空查询态之间缺少明确的视觉切换。

#### Scenario: 搜索结果页顶部保持搜索头与类型切换栏稳定
- **WHEN** 用户已经进入搜索结果态
- **THEN** 顶部继续展示包含返回键、当前查询词和搜索入口的搜索头
- **AND** 搜索头下方展示固定的结果类型切换栏
- **AND** 当前选中类型通过主强调色和指示线清晰表达
- **AND** 即使是 `MV` 这类短标签，结果类型项也保持与中文两字标签一致的宽度节奏和对齐基线

#### Scenario: 搜索结果行采用封面加双行文字的紧凑结构
- **WHEN** 用户查看歌曲、专辑、歌手或其他音乐结果列表
- **THEN** 每个结果行在左侧展示固定尺寸封面或等价缩略承载位
- **AND** 中间以标题在上、副标题在下的两行结构展示结果摘要
- **AND** 右侧展示独立的更多操作入口
- **AND** 结果列表整体保持稳定行高和连续滚动节奏

#### Scenario: 快速切换结果类型时当前 tab 不出现空白页
- **WHEN** 用户在搜索结果页快速点击或横滑切换不同结果类型
- **THEN** 当前选中的结果类型始终展示内容、局部 loading、空态或错误态之一
- **AND** 当前选中类型不会因为状态同步窗口短暂回落为无内容的空白页
- **AND** 已加载过的类型在切回时继续复用已有结果，而不是重新闪成空白
