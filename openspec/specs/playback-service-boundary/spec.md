## Purpose

定义后台播放服务的客户端边界，确保 `app` 与各业务页面通过稳定的播放客户端连接后台播放能力，而不是直接依赖播放服务实现细节。

## Requirements

### Requirement: 播放服务实现细节不得泄漏到 `app`
系统 SHALL 将后台播放服务的实现细节收敛在播放层模块内，`app` 不得直接依赖 `playback-service` 的实现类来完成播放控制或服务启动。

#### Scenario: 主壳与播放器页不直接引用播放服务实现类
- **WHEN** `app` 中的主壳、minibar、详情页或播放器页需要连接播放能力
- **THEN** 调用方通过稳定的播放客户端边界完成连接
- **AND** 调用方不直接引用 `com.wxy.playerlite.playback.process.*` 下的实现类

#### Scenario: 播放服务实现可在不改宿主业务代码的前提下演进
- **WHEN** 播放服务宿主类名、实现包路径或内部初始化方式发生调整
- **THEN** 宿主业务代码不需要同步修改为新的实现细节
- **AND** 播放层对外暴露的客户端边界保持稳定

### Requirement: 播放客户端边界继续覆盖现有核心播放链路
系统 MUST 通过稳定的播放客户端边界继续承载当前核心播放链路，保证结构收口不改变既有播放行为。

#### Scenario: 结构收口后核心播放操作仍可用
- **WHEN** 用户执行播放、暂停、恢复、切歌、打开播放列表或预热播放连接
- **THEN** 系统继续通过播放客户端边界完成这些操作
- **AND** 现有页面上的播放体验与行为语义保持一致

#### Scenario: 详情页和首页的播放入口继续共享同一播放真相源
- **WHEN** 用户从首页 minibar、详情页或播放器页发起播放控制
- **THEN** 系统仍连接到同一后台播放会话
- **AND** 不会因为边界收口导致播放状态出现多真相源分叉

### Requirement: 播放客户端边界支持下发当前 source config
系统 SHALL 通过稳定的播放客户端边界下发当前 source config，而不是要求 `app` 直接依赖播放服务实现细节来切换音源。

#### Scenario: 设置页通过播放客户端边界切换当前音源
- **WHEN** 用户在设置页把某个 source 设为当前音源
- **THEN** `app` 通过稳定的播放客户端边界下发该 source config
- **AND** 调用方不需要直接引用播放服务实现类或进程内实现细节

#### Scenario: source config 非法时边界返回明确失败
- **WHEN** `app` 通过播放客户端边界下发一个无法解析或不受支持的 source config
- **THEN** 播放客户端边界向调用方返回明确失败
- **AND** 调用方不需要感知播放服务内部的异常类型或构造方式

### Requirement: 播放服务重建后恢复当前 source config
系统 MUST 在播放服务重建或客户端重连后恢复最近一次成功生效的当前 source config，确保播放真相源保持稳定。

#### Scenario: 播放服务重建后继续恢复当前音源
- **WHEN** 播放服务因进程重建、连接断开或服务销毁后重新启动
- **THEN** 系统恢复最近一次成功生效的当前 source config
- **AND** 后续在线播放仍使用该 source 对应的运行时能力

#### Scenario: 客户端重连后不需要重新拼接播放服务实现细节
- **WHEN** `app` 与播放服务重新建立连接
- **THEN** 客户端仍通过既有边界读取和控制当前播放会话
- **AND** 不需要因为 source runtime 的引入而改为直接操作播放服务实现类
