## Purpose

定义用户会话、登录入口、用户资料读取与账户入口展示规则，确保账号能力在前台可恢复、可观察、可复用。

## Requirements

### Requirement: 应用提供独立的登录入口用于建立在线播放用户会话
系统 SHALL 提供独立的 `LoginActivity` 作为在线播放能力的登录入口，并通过 `user` module 统一处理手机号/邮箱登录请求、结果反馈与会话建立。

#### Scenario: 未登录用户进入登录页
- **WHEN** 用户从在线播放相关入口触发登录流程
- **THEN** 系统打开独立的 `LoginActivity`，而不是把登录逻辑直接耦合在播放器主页面中

#### Scenario: 未登录启动应用时可先进入登录页并允许跳过
- **WHEN** 应用完成会话恢复后确认当前未登录
- **THEN** 系统可以优先展示独立的 `LoginActivity`
- **AND** 用户可以点击“跳过”关闭登录页并继续进入主界面使用本地播放能力

#### Scenario: 手机登录成功后建立用户会话
- **WHEN** 用户在 `LoginActivity` 中完成手机登录且接口返回成功
- **THEN** 系统通过 `user` module 建立当前用户会话，并将结果暴露给前台 UI 与后续在线能力使用

#### Scenario: 邮箱登录成功后建立用户会话
- **WHEN** 用户在 `LoginActivity` 中切换到邮箱登录并提交有效邮箱与密码且接口返回成功
- **THEN** 系统通过 `user` module 建立当前用户会话，并将结果暴露给前台 UI 与后续在线能力使用

#### Scenario: 手机登录失败时保持现有本地能力可用
- **WHEN** 手机登录接口返回失败、参数错误或网络异常
- **THEN** 系统提示登录失败原因，并且不影响本地播放与已有本地播放列表能力

#### Scenario: 登录页允许切换手机号和邮箱登录
- **WHEN** 用户进入 `LoginActivity`
- **THEN** 系统展示手机号与邮箱两种登录方式切换入口
- **AND** 切换登录方式时只切换当前必填字段与提交行为，不影响“跳过”与本地播放可用性

#### Scenario: 欢迎页式登录布局保留轻量跳过入口
- **WHEN** 未登录用户打开 `LoginActivity`
- **THEN** 系统展示浅色音乐产品风欢迎页式布局
- **AND** 在右上角提供文字“跳过”入口
- **AND** 将主视觉、标题、副标题与登录操作区整体居中展示

### Requirement: 用户域定义稳定的 UserInfo 模型
系统 SHALL 在 `user` module 中定义稳定的 `UserInfo` 领域模型，并基于“手机登录返回结果 + 用户详情返回结果”的稳定字段并集做统一映射；调用方 MUST 不直接依赖原始 JSON 结构。

#### Scenario: 登录响应映射基础用户信息
- **WHEN** 手机登录接口返回用户账号与资料信息
- **THEN** 系统将至少 `userId`、`accountId`、`nickname`、`avatarUrl`、`vipType` 与当前登录相关账号标识映射到统一 `UserInfo` 模型

#### Scenario: 用户详情补充扩展字段
- **WHEN** 系统在已登录状态下请求用户详情接口成功
- **THEN** 系统使用详情返回结果补充或刷新 `UserInfo` 中的扩展字段，并保持字段来源一致

#### Scenario: 调用方通过领域模型读取用户数据
- **WHEN** UI、在线播放入口或其他业务能力需要读取当前用户信息
- **THEN** 系统返回统一的 `UserInfo` 模型，而不是暴露原始登录响应或原始详情响应

#### Scenario: 主界面账户入口在已登录时展示真实头像
- **WHEN** 系统已恢复有效用户会话且当前用户资料中存在头像地址
- **THEN** 主界面右上角账户入口显示圆形在线头像
- **AND** 点击行为仍保持为当前登录/账户入口

#### Scenario: 头像缺失或加载失败时回退默认人像
- **WHEN** 当前用户资料缺少头像地址，或头像资源加载失败
- **THEN** 主界面右上角账户入口回退显示默认人像图标

### Requirement: 用户中心页成为用户会话的主承载位
系统 SHALL 将用户中心页作为当前用户会话、登录入口和基础账户信息的主承载位，而不是继续只依赖首页顶部轻量入口承接用户相关交互。

#### Scenario: 已登录用户在用户中心页查看账户信息
- **WHEN** 系统已恢复有效用户会话且用户进入用户中心页
- **THEN** 系统展示当前用户的头像、昵称与基础账户信息
- **AND** 这些展示基于现有 `UserInfo` / 会话状态投影，而不是重新定义一套独立用户状态

#### Scenario: 收藏歌单等扩展内容不要求在首版完成
- **WHEN** 首版用户中心页面交付
- **THEN** 系统以头像、昵称和基础账户信息为主
- **AND** 收藏歌单等扩展内容可以在后续版本继续补充

#### Scenario: 未登录用户在用户中心页触发登录
- **WHEN** 当前没有有效用户会话且用户进入用户中心页
- **THEN** 系统展示明确的登录引导
- **AND** 用户可以从用户中心页打开现有 `LoginActivity`

#### Scenario: 首页概览态不再展示头像主入口
- **WHEN** 用户位于首页概览态
- **THEN** 系统不再在首页右上角展示头像主入口
- **AND** 用户相关主入口收口到用户中心页

### Requirement: 用户中心页与登录页对用户会话状态保持一致
系统 MUST 让用户中心页和登录页共享同一套用户会话状态；当登录态变化时，各前台承载位 SHALL 一致更新。

#### Scenario: 登录成功后用户中心页同步刷新
- **WHEN** 用户完成登录并返回主应用
- **THEN** 用户中心页展示已登录状态与对应用户资料

#### Scenario: 退出登录后用户中心页降级为未登录
- **WHEN** 用户主动退出登录或系统检测到会话失效
- **THEN** 用户中心页同步降级为未登录状态
- **AND** 后续用户相关入口重新要求用户登录

### Requirement: 用户会话与用户资料支持持久化恢复
系统 SHALL 持久化保存当前用户会话与必要的 `UserInfo` 快照，并在应用重启后恢复当前登录状态。

#### Scenario: 已登录状态下重启应用恢复会话
- **WHEN** 用户在已登录状态下关闭并重新启动应用
- **THEN** 系统恢复最近一次有效的用户会话与必要用户资料，而不是强制用户重新登录

#### Scenario: 本地会话缺失时恢复为未登录
- **WHEN** 应用启动时没有可用的本地用户会话
- **THEN** 系统将当前用户状态恢复为未登录，并等待用户显式发起登录

#### Scenario: 退出登录后清理本地会话
- **WHEN** 用户主动执行退出登录
- **THEN** 系统清空持久化的用户会话与用户资料快照，并将当前状态更新为未登录

### Requirement: 系统支持用户详情读取、退出登录与登录状态监测
系统 SHALL 支持在已登录状态下获取用户详情、主动退出登录，并在会话失效时统一降级为未登录状态。

#### Scenario: 已登录状态下读取用户详情
- **WHEN** 当前用户会话有效且系统请求用户详情
- **THEN** 系统返回当前用户的详情信息并刷新本地 `UserInfo`

#### Scenario: 关键接口返回未登录时统一降级
- **WHEN** 用户详情接口或其他受登录态约束的关键接口返回明确的未登录/失效结果
- **THEN** 系统将当前状态统一降级为未登录，并清理失效的用户会话

#### Scenario: 主动退出登录后状态同步更新
- **WHEN** 用户执行退出登录
- **THEN** 系统同步更新前台用户状态、清理本地会话，并使后续在线播放入口重新要求登录
