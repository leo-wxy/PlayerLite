## Purpose

定义独立本地歌曲页面的缓存展示、重新扫描和基于当前列表发起播放的能力，确保个人中心提供稳定的本地列表测试入口。

## Requirements

### Requirement: 本地歌曲页优先展示缓存结果并支持手动重新扫描
系统 MUST 提供一个独立的本地歌曲页面；该页面在已有缓存时应优先展示缓存结果，并通过右上角扫描入口支持手动重新扫描。

#### Scenario: 再次打开本地歌曲页时先展示缓存
- **WHEN** 用户此前已经完成过一次本地歌曲扫描，并再次打开本地歌曲页
- **THEN** 系统优先展示上一次缓存下来的本地歌曲结果
- **AND** 不要求用户等待重新扫描完成后才能看到列表

#### Scenario: 右上角扫描入口用于刷新本地歌曲结果
- **WHEN** 用户位于本地歌曲页并点击右上角扫描入口
- **THEN** 系统重新扫描本机音频媒体库
- **AND** 扫描成功后更新当前列表与本地缓存

### Requirement: 本地歌曲页支持基于当前列表的播放入口
系统 MUST 允许用户在本地歌曲页通过“播放全部”或点击单曲直接发起当前列表播放，并沿用当前播放列表替换与 index 激活语义。

#### Scenario: 点击播放全部后以当前本地歌曲列表开始播放
- **WHEN** 用户位于本地歌曲页并点击“播放全部”
- **THEN** 系统使用当前本地歌曲列表替换当前播放列表
- **AND** 以首曲作为激活项开始播放
- **AND** 播放请求成功后回到主壳层并展开播放器

#### Scenario: 点击某首本地歌曲后按目标 index 开始播放
- **WHEN** 用户位于本地歌曲页并点击某一首歌曲
- **THEN** 系统使用当前本地歌曲列表替换当前播放列表
- **AND** 以被点击歌曲对应的 index 作为激活项开始播放
- **AND** 播放请求成功后回到主壳层并展开播放器

#### Scenario: 本地歌曲播放请求失败时保留当前页并提示
- **WHEN** 用户位于本地歌曲页并点击“播放全部”或某一首歌曲，但播放请求未成功发起
- **THEN** 系统保持当前本地歌曲页可见
- **AND** 通过短提示告知用户播放启动失败

### Requirement: 本地歌曲页接入统一歌曲对象体系并提供详情入口
系统 SHALL 让本地歌曲页中的歌曲接入统一歌曲对象体系，但继续保留“点击即播放”为主路径；本地歌曲页 MUST 使用统一的三点更多入口承载单曲次级动作，而不是继续让本地歌曲使用独立的详情图标作为唯一次级入口。

#### Scenario: 点击本地歌曲主区域仍然直接播放
- **WHEN** 用户位于本地歌曲页并点击某一首本地歌曲的主区域
- **THEN** 系统继续使用当前本地歌曲列表替换当前播放列表
- **AND** 以被点击歌曲对应的 index 作为激活项开始播放

#### Scenario: 本地歌曲通过统一更多入口承载次级动作
- **WHEN** 用户位于本地歌曲页并点击某一首本地歌曲右侧的三点更多入口
- **THEN** 系统展示统一的单曲更多菜单
- **AND** 菜单至少提供“下一首播放”“查看歌曲详情”
- **AND** 系统不展示依赖在线歌手或专辑目标标识的动作

#### Scenario: 本地歌曲列表不再使用独立详情图标
- **WHEN** 用户浏览本地歌曲页中的单曲列表
- **THEN** 列表右侧不再展示仅用于进入详情页的独立 Info 图标
- **AND** 单曲次级动作统一由三点更多入口承载

#### Scenario: 本地歌曲详情页不伪造在线能力
- **WHEN** 用户进入本地歌曲详情页
- **THEN** 系统不展示依赖在线歌曲标识的收藏、百科或查看在线专辑/歌手动作
- **AND** 页面仍保持稳定的详情承载结构
