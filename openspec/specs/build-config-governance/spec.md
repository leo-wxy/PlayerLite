## Purpose

定义项目共享 Android 构建约束与运行时环境配置的集中治理方式，避免跨模块重复配置和业务装配代码中的环境硬编码继续扩散。

## Requirements

### Requirement: 共享 Android 构建约束集中治理
系统 SHALL 将跨模块共享的 Android 构建约束集中治理，使模块脚本仅保留确有差异的配置，而不是继续在各模块中重复拷贝相同设置。

#### Scenario: 调整共享 Android 配置时不需要逐模块重复修改
- **WHEN** 项目需要统一调整 `compileSdk`、`minSdk`、Java 版本、Compose 开关或测试基础配置
- **THEN** 开发者通过集中治理入口完成修改
- **AND** 不需要逐个模块手工同步相同配置

#### Scenario: 模块脚本只保留自身差异项
- **WHEN** 开发者查看任意模块的 Gradle 脚本
- **THEN** 共享配置通过统一约束继承
- **AND** 模块脚本主要表达命名空间、特有依赖、native 配置或其他模块级差异

### Requirement: 运行时环境配置来自统一配置源
系统 MUST 将运行时环境相关配置从业务装配代码中收口到统一配置源，而不是继续硬编码在通用装配类中。

#### Scenario: 切换后端环境不需要修改业务装配代码
- **WHEN** 项目需要调整 API base URL 或其他环境相关配置
- **THEN** 开发者通过统一配置源完成切换
- **AND** 不需要直接修改业务装配代码中的硬编码常量

#### Scenario: 宿主装配代码只消费配置而不定义环境真相
- **WHEN** 应用构建依赖装配对象
- **THEN** 装配代码读取统一配置源中的环境值
- **AND** 装配代码不再成为运行时环境参数的硬编码真相源
