## Purpose

定义在线与本地播放条目投影到统一共享播放契约的边界，以及播放开始后队列元数据如何按稳定 item id 异步回填，确保 app 与播放服务围绕同一份跨进程可播放数据协作。

## Requirements

### Requirement: 在线与本地模型投影到统一共享播放契约
系统 SHALL 允许在线 `MusicInfo` 与本地 `LocalMusicInfo` 投影到同一种最小共享播放契约，以便 app 与播放服务围绕同一份可播放数据协作，而不是让播放服务直接依赖在线专属模型。

#### Scenario: 在线歌曲投影到共享播放契约
- **WHEN** app 侧准备把一首在线歌曲同步到播放服务
- **THEN** 系统将在线 `MusicInfo` 投影为统一共享播放契约
- **AND** 共享契约保留 `songId`、标题、歌手、时长、封面、请求头与必要上下文字段
- **AND** 当上游具备稳定专辑目标标识时，系统同时保留 `albumId`

#### Scenario: 本地歌曲投影到同一共享播放契约
- **WHEN** app 侧准备把一首本地歌曲或 `content://` 音源同步到播放服务
- **THEN** 系统将本地领域模型投影到同一共享播放契约
- **AND** 播放服务不要求本地歌曲携带在线专属字段才能完成播放

### Requirement: 上游详情字段必须映射为语义化共享元数据
系统 SHALL 通过显式 mapper 把上游详情接口字段转换为语义化共享元数据，而不是把 `id`、`name`、`ar`、`al`、`dt` 这类原始缩写直接暴露为共享播放契约。

#### Scenario: `/song/detail` 原始字段被转换为语义化播放字段
- **WHEN** 系统消费 `/song/detail` 返回的歌曲详情
- **THEN** 它将 `id` 映射为 `songId`
- **AND** 将 `name`、`ar`、`al`、`dt` 等字段映射为标题、歌手、专辑与时长等语义化字段
- **AND** 共享播放契约不直接暴露这些原始缩写字段

### Requirement: 队列元数据允许在播放开始后按 item id 异步回填
系统 SHALL 允许播放列表先凭稳定 `songId` 建立并开始播放，然后再按稳定 item id 异步回填队列元数据，而不重建当前队列或改变激活项语义。

#### Scenario: 异步回填元数据不会改变当前队列顺序
- **WHEN** 当前播放列表已经建立并开始播放
- **AND** 后台随后拿到了同一批歌曲更完整的标题、歌手、专辑、封面或时长
- **THEN** 系统按稳定 item id 更新已有队列项的元数据
- **AND** 当前队列顺序与激活项保持不变

#### Scenario: 播放器页可直接使用异步回填后的封面与时长
- **WHEN** 当前曲目的共享快照和队列项已经拥有封面与时长
- **THEN** 播放器页直接使用这些共享字段刷新主视觉和时长展示
- **AND** 不要求再单独发起一次详情查询

#### Scenario: 队列项为在线歌曲时保留稳定专辑目标标识
- **WHEN** app 侧已为某一首在线歌曲建立共享播放队列项
- **AND** 上游已经提供该歌曲稳定的专辑目标标识
- **THEN** 队列项保留该 `albumId`
- **AND** 当前播放队列中的“查看专辑”动作直接复用这一稳定元数据

### Requirement: 共享队列项保留可刷新在线资源上下文
系统 SHALL 在共享队列项中保留在线歌曲稳定身份和重新解析播放资源所需的最小上下文，使搜索、详情、最近播放和恢复链路都能在没有长期直链时重新解析当前歌曲。

#### Scenario: 搜索结果投影为可恢复在线队列项
- **WHEN** 用户点击搜索结果中的在线歌曲开始播放
- **THEN** 系统将该歌曲投影为包含稳定 `songId`、标题、歌手、专辑、封面、时长和必要 source 上下文的共享队列项
- **AND** 播放准备链路可以基于该队列项重新解析真实播放地址

#### Scenario: 队列恢复不依赖旧直链
- **WHEN** app 或播放服务从本地恢复出一条在线歌曲队列项
- **AND** 该队列项保存的旧播放地址缺失或已经过期
- **THEN** 系统仍可基于稳定歌曲身份和 source 上下文重新解析播放资源

### Requirement: 单曲来源上下文随当前队列项保存但不污染长期资产
系统 SHALL 允许当前队列项记录单曲来源 override 或实际生效 source，用于当前播放、失败重试和恢复；该信息 MUST 不自动写回收藏、歌单或搜索结果源数据。

#### Scenario: 备用 source 成功后更新当前队列项上下文
- **WHEN** 当前在线歌曲通过备用 source 兜底成功播放
- **THEN** 当前队列项可记录本次实际生效 source
- **AND** 后续恢复当前队列项时可以优先使用该单曲来源上下文

#### Scenario: 长期歌单不被单曲来源切换污染
- **WHEN** 用户对当前队列中的某首歌执行单曲来源切换
- **THEN** 系统只更新当前队列项或当前会话上下文
- **AND** 该歌曲所属的收藏、歌单、最近播放源数据不会被静默改写
